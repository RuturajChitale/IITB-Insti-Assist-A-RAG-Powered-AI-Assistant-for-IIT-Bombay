# 🎓 IITB Insti-Assist — Academic Assistant

**🌐 Live Demo:** https://iitb-insti-assist-a-rag-powered-ai-assistant-for-iit-bombay-id.streamlit.app/

A Retrieval-Augmented Generation (RAG) assistant that answers questions about
**IIT Bombay's academic rules** — grading, CPI/SPI, course registration
categories, examination rules, and the academic calendar — grounded in the
Institute's own official documents. It refuses to answer ("I don't know")
when a question isn't covered by those documents, instead of hallucinating.

Built for the **Final Project of WnCC Learners' Space — Machine Learning
(LS-26)**.

---

## How it works

```text
data/raw/*.pdf  →  ingest.py  →  data/processed/chunks.jsonl
                        │
                        ▼
                 build_index.py  →  FAISS vector index (all-MiniLM-L6-v2 embeddings)
                        │
User question  →  retriever.py  (top-k similarity search)
                        │
                        ▼
              rag/pipeline.py — groundedness gate
                        │
                        ▼
                 rag/llm.py — Groq (Llama) generates an answer
                              using ONLY the retrieved context
                        │
                        ▼
                   app.py — Streamlit chat UI, shows sources
```

**Scope chosen:** Academic Assistant (course registration, grading policy,
academic calendar, exam rules) — see `WRITEUP.md` for why.

## Project structure

```text
iitb-insti-assist/
├── app.py                     # Streamlit UI
├── requirements.txt
├── .env.example               # copy to .env and add your Groq API key
├── rag/
│   ├── config.py              # paths, model names, prompt, source list
│   ├── retriever.py           # FAISS load + top-k semantic search
│   ├── llm.py                 # Groq chat completion call
│   └── pipeline.py            # retrieve -> groundedness check -> generate
├── scripts/
│   ├── download_sources.py    # downloads the 6 official IITB PDFs
│   ├── ingest.py              # PDF -> cleaned, overlapping text chunks
│   └── build_index.py         # embeds chunks + builds the FAISS index
├── data/
│   ├── raw/                   # downloaded PDFs (gitignored, regenerated)
│   └── processed/             # chunks.jsonl + faiss.index (gitignored, regenerated)
└── WRITEUP.md                 # 2-page project write-up
```

The PDFs and the built index are **not committed to git** — they're
regenerated by running the three setup scripts below, so the repo always
ingests the current official documents rather than a possibly stale copy.

## Setup

### 1. Clone and install dependencies

```bash
git clone <your-repo-url>
cd iitb-insti-assist
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Get a free Groq API key

Sign up at https://console.groq.com/keys and create an
API key (Groq's free tier is generous and doesn't hit quota walls the way
Gemini's free tier does).

```bash
cp .env.example .env
# then edit .env and paste your key into GROQ_API_KEY=
```

### 3. Build the knowledge base (one-time)

```bash
python scripts/download_sources.py   # downloads 6 official IITB academic PDFs
python scripts/ingest.py             # parses + chunks them
python scripts/build_index.py        # embeds chunks + builds the FAISS index
```

If a download fails (e.g. a URL changed), the script tells you which file to
grab manually — just save it as `data/raw/<id>.pdf` using the `id` from
`rag/config.py::SOURCES` and re-run `ingest.py` + `build_index.py`.

### 4. Run the app

```bash
streamlit run app.py
```

Open the local URL Streamlit prints (usually `http://localhost:8501`).

## Data sources

All 6 documents are official, publicly published IIT Bombay academic
documents (see `rag/config.py::SOURCES` for the live links):

1. UG Rule Book (B.Tech/B.S./Dual Degree Rules & Regulations)
2. Ph.D. Programme Rules & Regulations
3. M.Tech./M.Phil./Dual Degree PG Rules & Regulations
4. M.Eng. Programme Rules
5. Academic Calendar 2025–26
6. Grading Policy Summary

## Known limitations

See `WRITEUP.md` for a fuller discussion. In short: PDF text extraction can
mangle tables (e.g. multi-column grade tables), the similarity-threshold
groundedness gate is a heuristic rather than a true confidence score, and
the assistant only knows what's in these 6 documents — it has no knowledge
of hostel life, clubs, placements, or anything outside the Academic
Assistant scope.
